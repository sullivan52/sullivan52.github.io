Stock Shark — Android (Java, SQLite)

Summary: Stock Shark is an Android inventory management app for small businesses, warehouses, and individual users to track stock levels. Originally built in CS 360 (Mobile Architecture and Programming), this enhanced version emphasizes security, usability, and maintainability.

Code

  • Enhanced (browse): github.com/sullivan52/stock-shark-android

Narrative

Overview

The artifact is an Android inventory management application, originally created in CS 360 earlier this year. It allows users to add items, adjust quantities, and review current stock. The enhancement effort modernized the codebase, improved usability, and strengthened security.

Why it’s in the ePortfolio

I selected this artifact because it is my strongest demonstration of software design and engineering to date. It reflects thoughtful use of Material Design and UI/UX principles and shows that I can deliver professional-quality mobile experiences—an area I’m excited to pursue.

Enhancements Made

  • Modularized responsibilities: clearer separation of concerns, focused classes, and reusable utilities.
  • Secured database operations: scoped SQLite queries to users; parameterized access.
  • Secure coding practices: password hashing, input validation, and basic SQLi prevention.
  • Material Design compliance: updated components and consistent patterns.
  • Expanded XML layouts & drawables: new screens, icons, and assets to improve navigation and branding.
  • Robust error handling: consistent messages, defensive checks, and fail-safes.
  • Accessibility: content descriptions, real-time validation, improved focus handling.
  • UX polish: clearer hierarchy, feedback loops, toasts, and real-time guidance.
  • Documentation: JavaDoc across core methods; clearer business logic and error behavior.
  • Android best practices: externalized strings/dimensions for localization and theming.
  • Debugging & testing workflow: emulator-based reproduction, structured logging, targeted fixes.

Outcomes Addressed

  • Security mindset: security built into design (hashing, validation, scoped queries, error hygiene).
  • Professional communication: comprehensive JavaDoc and improved internal documentation to aid future contributors.
  • Software engineering & design: maintainable architecture, modular code, and accessible UI.

Reflection

Enhancing the app showed me how many moving parts go into a production-ready mobile application. Layering accessibility and security onto a simple codebase surfaced real-world challenges—from Material theming conflicts and app freezes to cryptic exceptions like the “1.0dip” NumberFormatException. Working through these issues sharpened my debugging workflow and reinforced thinking from both user and developer perspectives. The result is a codebase that’s easier to understand, safer to extend, and more pleasant to use. This project strengthened my confidence in mobile development and confirmed my interest in roles that blend creativity, problem-solving, and user empathy.
