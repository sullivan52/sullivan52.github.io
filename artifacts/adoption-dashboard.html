Animal Adoption Dashboard — Python/Dash, MongoDB

Summary: Refactored a CS-340 shelter tool into a public-facing adoption dashboard with fast filtering, cleaner visuals, and pragmatic security. The project moves from a Jupyter prototype to a modular Python/Dash app that can handle large datasets and surface the right animals quickly.

Code

  • Enhanced (browse): github.com/sullivan52/animal-rescue-dashboard

Narrative

Overview

Originally built for CS-340 (Client/Server Development), the first version lived in a Jupyter notebook and targeted a fictional client (Grazioso Salvare) to find rescue dogs using MongoDB and simple charts/maps. The enhanced version broadens the scope into a full adoption dashboard, letting the public filter by species, breed, age, sex, and other attributes to find a good match.

Why it’s in the ePortfolio

This artifact shows breadth: data ingestion at scale, a realistic UI in Dash, secure configuration, and performance-minded filtering. It demonstrates taking a narrowly scoped academic artifact and evolving it into an industry-relevant tool focused on user needs.

Enhancements Made

  • Architecture & Security: migrated from notebook to modular Python app; moved credentials to environment variables; added input validation and error handling.
  • User Experience: redesigned the interface from a rescue-specific tool to a general adoption platform; modern filters (dropdowns, sliders, quick-select); cohesive visual design.
  • Data & Performance: integrated CSV processing for 10k+ records; optimized load/transform/display paths; clearer labeling and readable formats.
  • Visualization: streamlined the map toward adopter-focused info; removed noisy charts; added toggles and richer popups.
  • Code Quality: separated concerns into modules; replaced hardcoded values with constants/config; improved maintainability with reusable functions and documentation.

Outcomes Addressed

  • Implement value-driven solutions: turned a specialized classroom tool into a practical adoption app aligned with real users.
  • Sound computing practices: structured modules, secure credential management, and systematic error handling.
  • Professional communication: README and in-code docs clarify setup, data flows, and design decisions.

Reflection

This work grew both my engineering and design thinking. On the technical side, I learned Dash, modular Python patterns, and how to wrangle real-world data at scale. The hardest part was re-architecting the UX and data flow—replacing hardcoded rescue logic with flexible filters while keeping the app fast and simple. The biggest takeaway: iterate with users in mind, and measure success by clarity, not feature count.
